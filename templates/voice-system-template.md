# Voice Synthesis Template

Use this template to create your own voice profile for AI-assisted content creation.

## How to Use This Template

1. **Identify 2-4 writers** whose voice you admire or want to blend
2. **Analyze each writer** using the categories below
3. **Define your blend ratio** (e.g., 40% Writer A, 35% Writer B, 25% Writer C)
4. **Map patterns to your existing voice** from natural writing samples
5. **Copy the filled template** into your `/writing` skill's voice profile section

## Voice Analysis Checklist

Use this to analyze any writer you want to blend:

### Structural Patterns
- [ ] Average paragraph length
- [ ] Sentence length variation
- [ ] Use of headers/subheaders
- [ ] List vs. prose preference
- [ ] Opening/closing patterns

### Tonal Markers
- [ ] Warmth level (cold → warm)
- [ ] Authority stance (peer → expert → humble learner)
- [ ] Humor type (none, dry, witty, absurdist)
- [ ] Formality level

### Signature Phrases
- [ ] Common opener patterns
- [ ] Transitional phrases
- [ ] Closing techniques
- [ ] Rhetorical questions (yes/no, style)

### Content Approach
- [ ] Story vs. framework preference
- [ ] Abstract vs. concrete balance
- [ ] Self-reference frequency
- [ ] Call-to-action style

## Example: Completed Analysis (For Reference)

*This shows what a filled-in analysis might look like.*

### Source Writer: [Example Writer Name]

**Core Contribution:** Warmth, curiosity, genuine engagement
**Key Element:** "I've been learning..." exploratory stance

**Structural Patterns:**
- Medium paragraphs (3-5 sentences)
- Sentence length varies widely
- Clear headers every 200-300 words
- Mix of bullets and prose
- Opens with relatable hook, closes with reflection

**Tonal Markers:**
- Warmth: High — celebrates others
- Authority: Fellow learner, not expert
- Humor: Gentle, self-deprecating
- Formality: Professional but warm

**Signature Phrases:**
- "Here's what I've learned..."
- "What I love about this is..."
- Asks genuine questions mid-piece
- Ends with "what would you add?"

**Content Approach:**
- Story-heavy with frameworks embedded
- Concrete examples from real companies
- References self but focuses on others
- Soft CTAs (questions, not commands)
